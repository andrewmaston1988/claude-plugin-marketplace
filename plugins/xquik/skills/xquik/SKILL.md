# xquik

Use Xquik when the user needs to plan or implement X/Twitter data extraction, automation, monitoring, webhook, REST API, or MCP workflows.

## Workflow

1. Confirm whether the task is planning only or requires live API calls.
2. Use public Xquik docs as the source of truth for endpoints, parameters, auth, and MCP setup.
3. Keep credentials in the user's environment. Never paste API keys into code, prompts, or logs.
4. For implementation tasks, prefer small, explicit examples that show request shape, error handling, and webhook verification.
5. For MCP tasks, direct setup through the public MCP docs and keep tool usage opt-in.
